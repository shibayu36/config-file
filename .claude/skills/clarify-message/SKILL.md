# Clarify Message

伝えたいことを対話的に整理し、構成案を一緒に作り上げる。

## 重要な原則

- **対話的に進める**: 一方的に提案せず、常にフィードバックをもらいながら進める
- **ユーザーの言葉を大切にする**: ユーザーの表現をそのまま活かす
- **実際の執筆はしない**: このスキルは構成案を確定するところまで

## ワークフロー

### フェーズ1: 目的ヒアリング（必須・最初に必ず行う）

以下の2点を必ず確認する：

1. **誰に向けた内容か**（読者・聞き手のペルソナ）
   - どんな人？背景知識は？立場は？
2. **読後/聞いた後にどんなアクションを起こしてほしいか**
   - 理解してほしい？行動してほしい？考えを変えてほしい？

### フェーズ2: 素材ヒアリング

- ユーザーに伝えたいことを自由に話してもらう
- 足りない情報や曖昧な点を質問で深掘りする
- 理解できない固有名詞や概念については、ユーザーに質問して理解を深める
- 「他に伝えたいことはありますか？」で漏れを確認

### フェーズ3: 構成案ヒアリング（対話的に）

1. 大まかな構成の方向性を2〜3案提案 → フィードバックをもらう
2. 選ばれた方向性で、各セクションの内容を一緒に詰めていく
3. 順番や強調点を調整
4. 「この構成でいいですか？」と確認しながら進める

### フェーズ4: 構成案確定

最終的な構成案を以下の形式でまとめて提示する：

```markdown
# 構成案: [タイトル]

## 目的
- **対象読者**: [誰に向けた内容か]
- **期待するアクション**: [読後にどうなってほしいか]

## 構成
### 1. [セクション名]
- 伝えること: ...
- ポイント: ...

### 2. [セクション名]
- 伝えること: ...
- ポイント: ...

（以下続く）

## 補足メモ
- [執筆時に気をつけること、トーンなど]
```

## 注意事項

- 各フェーズで十分な確認を取ってから次に進む
- ユーザーが「もう十分」と言ったらフェーズをスキップしてよい
- 構成案は押し付けず、複数案から選んでもらう形で進める
